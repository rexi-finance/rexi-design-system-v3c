Decision Button Spec

  • Primary: blue-muted bg, blue border — recommended action, always first
  • Default: glass border, neutral bg — alternative actions
  • Destructive: red-muted hover, red icon — dangerous actions, always last
  • Layout: vertical stack, full-width buttons, 6px gap
  • Icons: 14px Lucide, left-aligned, semantic color per variant
  • One group per message. Once user clicks, buttons disable and show selected state

When the AI needs structured input, it renders an inline form with glass-styled inputs, selects, and action buttons. Replaces back-and-forth text conversations for data entry.

Adaptive Overflow Rules

  • Inline threshold: up to 5 rows / small KPI / compact chart → stays in chat bubble
  • Overflow trigger: >5 rows, complex charts, wide tables → compact preview + expand link
  • Overlay: bg-elevated, glass-border, radius-xl, deep shadow. Centered over main area.
  • Click "expand": opens overlay drawer with full content. Chat stays visible behind.
  • Backdrop: black 40% over main content (not over chat panel)
  • Close: X button or click backdrop

Rich Text Typography Scale

  • H2 (section title): 15px, 700, text-bright, -0.01em tracking — used for major sections
  • H3 (subsection): 13px, 600, text — used for content groups within a message
  • Body (paragraph): 13px, 400, text-muted, 1.75 line-height — key words bolded to text color
  • List items: 12px, text-muted, disc bullets via ::before pseudo — nested detail level
  • Code spans: 11px, JetBrains Mono, emerald on glass-bg-4 — entity names, identifiers
  • Blockquote: 12px italic, text-muted, 2px blue left border, glass-bg-1 — examples, evidence
  • Badges: 10px, 600, colored pill — risk levels (HIGH=red, MEDIUM=amber, LOW=emerald)
  • Stat row: inline mini-KPIs with overline label + 800 value — summary metrics
  • Divider: 1px glass-border-subtle, 14px vertical margin — section separator
