Surface Model — Floating Sidebar

Actualizado: sidebar flotante transparente como en el brandbook. Los iconos viven directamente sobre #1a1a1a. Todo comparte el mismo fondo — la profundidad viene de las glass layers, no de paneles sólidos.

Principio #3: Subtract, don't add. El sidebar no necesita su propio fondo — los iconos flotan. Menos capas = más limpio.

JERARQUÍA DE CAPAS (actualizada)
0
bg — Todo Vive Aquí
#1a1a1a
El lienzo único. Sidebar, content, stats focales — todo comparte este fondo. Es el espacio que respira (Principio #2).
1
bg-elevated — Modals, Overlays, Dropdowns
#1e1e1e
Solo para elementos que necesitan "flotar" sobre el contenido: modals, command palette, dropdown menus. NO para sidebar ni paneles inline.
2
glass — Cards Secundarias & Contenedores
gradient 135° (white 5% → 1% → 3%) + border white 6%
Métricas secundarias, formularios, paneles de contenido. El gradiente crea profundidad sutil.
3
glass-mid — Table Headers, Hovers, Active States
rgba(255,255,255, 0.06)
Feedback visual: hover de sidebar items, table headers, hover de rows.
4
glass-bright — Selected, Focused
rgba(255,255,255, 0.10)
Sidebar item activo, tab seleccionada, input con focus.
